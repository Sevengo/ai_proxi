# Отчет об анализе и исправлениях скриптов создания ярлыков

**Дата:** 03.03.2026  
**Репозиторий:** https://github.com/Sevengo/ai_proxi

## 🔍 Выявленные проблемы

### 1. Жестко заданные пути
**Проблема:** Все скрипты использовали хардкод пути `D:\tools\ai`  
**Последствия:** Скрипты не работали при копировании в другую директорию

**Найдено в файлах:**
- `create-ai-cli-shortcuts.ps1`
- `create-desktop-shortcuts.ps1`
- `create-shortcuts-new.ps1`
- `update-cli-shortcuts.ps1`
- `claude-interactive.ps1`
- `copilot-interactive.ps1`
- `launch-ai.ps1`
- `proxy.ps1`
- `test-providers.ps1`
- `powershell-aliases.ps1`

### 2. Отсутствие проверки существования файлов
**Проблема:** Ярлыки создавались без проверки существования целевых скриптов  
**Последствия:** Нерабочие ярлыки при отсутствии файлов

### 3. Предупреждение Windows Terminal
**Проблема:** При запуске появляется сообщение о профиле по умолчанию  
**Последствия:** Визуальная помеха, но не влияет на функциональность

## ✅ Внесенные исправления

### 1. Автоматическое определение пути

**Добавлено во все скрипты:**
```powershell
$scriptPath = $PSScriptRoot
if (-not $scriptPath) {
    $scriptPath = Split-Path -Parent $MyInvocation.MyCommand.Path
}
```

**Преимущества:**
- ✅ Скрипты работают из любой директории
- ✅ Автоматическое определение местоположения
- ✅ Портативность решения

### 2. Проверка существования файлов

**В функциях создания ярлыков добавлена проверка:**

**create-ai-cli-shortcuts.ps1:**
```powershell
# Проверяем существование скрипта launch-ai.ps1
$launchScript = Join-Path $toolsPath "launch-ai.ps1"
if (-not (Test-Path $launchScript)) {
    Write-Host "⚠️  Файл не найден: $launchScript" -ForegroundColor Red
    return
}
```

**create-desktop-shortcuts.ps1 и create-shortcuts-new.ps1:**
```powershell
# Проверяем существование целевого файла
if ($TargetPath -ne "notepad.exe" -and $Arguments -match '\.ps1"') {
    $scriptPath = $Arguments -replace '.*-File\s+"([^"]+)".*', '$1'
    if (-not (Test-Path $scriptPath)) {
        Write-Host "⚠️  Пропуск '$Name' - файл не найден: $scriptPath" -ForegroundColor Yellow
        return
    }
}
```

**update-cli-shortcuts.ps1:**
```powershell
# Проверяем существование скрипта
$scriptPath = Join-Path $toolsPath $Script
if (-not (Test-Path $scriptPath)) {
    Write-Host "⚠️  Пропуск '$Name' - файл не найден: $scriptPath" -ForegroundColor Yellow
    return
}
```

### 3. Информативный вывод

**Добавлено в вывод скриптов:**
```powershell
Write-Host "Scripts path: $toolsPath" -ForegroundColor Cyan
```

Теперь пользователь видит, откуда запущены скрипты.
